How to Setup Your Own Certificate Authority (on Windows)
--------------------------------------------------------

Assumptions:
 A) You want to setup your own Certificate Authority
 B) You have a Java JDK [that includes keytool]



Procedures
----------
 1. Download and install OpenSSL 1.0.1p with development files
    a. Go to http://www.openssl.org/related/binaries.html
    b. Select "Win64 OpenSSL v1.0.1p" (next to Works with MSVC++, Builder 3/4/5, and MinGW. Comes in form of self-install executables. )
       NOTE:  It should take you here:  http://slproweb.com/products/Win32OpenSSL.html
    c. Select Win64 OpenSSL v1.0.1p
       NOTE:  It should take you here:  http://slproweb.com/download/Win64OpenSSL-1_0_1p.exe         
    d. Save Win64OpenSSL-1_0_1p.exe to your c:\vault
       a) Run c:\vault\Win64OpenSSL-1_0_1p.exe
          1) In the "Welcome to the openSSL (64-bit) setup wizard" screen, press Next
          2) In the License Agreement screen, select "I accept" and press "Next"
          3) In the "Select destination location" screen
             Set the destination to c:\tools\openssl-1.0.1p
             Press Next
          4) In the "Select Start Menu Folder" screen, use the defaults and press Next
          5) In the "Select Additional Tasks", choose "The Openssl binaries /bin directory" and press Next
          6) In the "Ready to Install" screen, press Install
 
    e. Add C:\tools\openssl-1.0.1p\bin to your PATH
    
    f. Verify that openssl is in your path
       DOS> openssl version
       OpenSSL 1.0.1p 9 Jul 2015
  
  
  
 2. Setup you own Root Certificate Authority directories
    a. Open a DOS window by pressing <Start><Run>CMD
    b. Inside the DOS window, do this:
       CMD> rd /s /q c:\tools\root-ca
       CMD> mkdir    c:\tools\root-ca
       CMD> cd /d    c:\tools\root-ca
       CMD> mkdir conf
       CMD> mkdir private
       CMD> mkdir public
       CMD> mkdir signed-keys
       CMD> echo 01 > conf/serial
       CMD> type nul > conf/index
       CMD> echo unique_subject = no > conf/index.attr
  
  
  
  3. Create this file:  openssl.cnf
     Create an empty file here:  c:\tools\root-ca\conf\openssl.cnf
     NOTE:  Remove any leading spaces
     
     Copy this to your file located here:   c:\tools\root-ca\conf\openssl.cnf
     NOTE:  for the default_md, you can use sha1 or sha256
     
     
        [ req ]
        default_bits            = 4096
        default_keyfile         = ./private/root.pem
        default_md              = sha256
        prompt                  = no
        distinguished_name      = root_ca_distinguished_name
        x509_extensions = v3_ca
        
        [ root_ca_distinguished_name ]
        countryName             = US
        stateOrProvinceName     = Virginia
        localityName            = Merrifield
        organizationName        = Example Inc Root
        commonName              = Example Inc Root CA
        emailAddress            = adam@zztop.com
        
        [ v3_ca ]
        subjectKeyIdentifier=hash
        authorityKeyIdentifier=keyid:always,issuer:always
        basicConstraints = CA:true
        
        [ ca ]
        default_ca              = CA_default
        
        [ CA_default ]
        dir                     = .
        new_certs_dir           = ./signed-keys
        database                = ./conf/index
        certificate             = ./public/root.pem
        serial                  = ./conf/serial
        private_key             = ./private/root.pem
        x509_extensions         = usr_cert
        name_opt                = ca_default
        cert_opt                = ca_default
        default_crl_days        = 30
        default_days            = 365
        default_md              = sha1
        preserve                = no
        policy                  = policy_match
        
        [ policy_match ]
        countryName             = optional
        stateOrProvinceName     = optional
        organizationName        = optional
        organizationalUnitName  = optional
        commonName              = supplied
        emailAddress            = optional
        
        [ usr_cert ]
        basicConstraints=CA:FALSE
        subjectKeyIdentifier=hash
        authorityKeyIdentifier=keyid,issuer:always
        nsCaRevocationUrl     = https://www.example.com/example-ca-crl.pem
     
    
    NOTE:  At this point, you should have these files:
        c:\tools\root-ca\conf\index
        c:\tools\root-ca\conf\index.attr
        c:\tools\root-ca\conf\openssl.cnf
        c:\tools\root-ca\conf\serial
        c:\tools\root-ca\private\
        c:\tools\root-ca\public\        
        c:\tools\root-ca\signed-keys\



 4. Verify that your c:\tools\root-ca\conf\serial holds this [with no quotes]
    01
     
 
 
 5. Verify that your c:\tools\root-ca\conf\index.attr holds this  [with no quotes]
    unique_subject = no
    
      
  
 6. Generate the public and private Root CA keys
    CMD> cd /d c:\tools\root-ca
    CMD> openssl req -nodes -config conf/openssl.cnf -days 1825 -x509 -newkey rsa:2048 -out public/root.pem -outform PEM
         Loading 'screen' into random state - done
         Generating a 2048 bit RSA private key
         .............++++++
         ........++++++
         unable to write 'random state'
         writing new private key to './private/root.pem'
         -----
    
    Explanation of the command options
    •   The –nodes tells openssl to not encrypt the private key.  This is useful when your web server starts automatically at boot time.  If the private key is encrypted, you will be required to enter the password every time your web server is restarted.
    •   The –newkey rsa:2048 generates a new private key of type RSA of length 2048 bytes
    •   The –days 1825 makes the certificate valid for 1825 days
    •   The –x509 output a self-signed certificate instead of a certificate request

 9. Distribute your Root Public Certificate (import it into your web browser)
    a. Give the c:\tools\root-ca\public\root.pem to teammates



10. Import the public root.pem into your firefox web browser
    a. Startup Firefox
    b. Pull Tools / Options / Advanced
    c. Click "View Certificates"
    d. Click "Authorities" tab
    e. Click "Import"
    f. Browse to where you have the c:\tools\root-ca\public\root.pem
    g. Check “Trust this CA to identify websites"
       Check "Trust this CA to identify email users"
       Check "Trust this CA to identify software developers"
       Press OK
       Press OK
